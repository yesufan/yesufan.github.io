二、常用命令

完整命令

  1. hexo new [layout] 创建一篇新文章。如果没有layout提供，
  2. Hexo将使用default_layoutfrom _config.yml。如果title包含空格，请用引号将其包围。
  3. hexo generate 生成静态目录到public目录
  4. hexo server 开启预览访问端口（(默认端口4000，’ctrl+c’关闭server)
  5. hexo deploy # 项目部署
  6. hexo help # 查看帮助
  7. hexo version # 查看Hexo的版本
  8. hexo publish [layout]
  9. hexo list < type > #列出所有路线
  10. hexo migrate < type >#迁移其他博客系统的内容。

简写命令

  • -d， –deploy 一代完成后部署

    -w， –watch 观看文件更改

    -b， –bail 如果在生成期间抛出任何未处理的异常，则引发错误

    -f， –force 强制再生

    -p， –port 覆盖默认端口

    -s， –static 只提供静态文件

    -l， –log 启用记录器。覆盖记录器格式。

    -g， –generate 在部署之前生成

三、搭建网站遇到的坑

1.hexo生成博文插入图片不显示

配置文件

把主页配置文件_config.yml 里的post_asset_folder:这个选项设置为true

在你的hexo目录下执行这样一句话npm install hexo-asset-image --save，这是下载安装一个可以上传本地图片的插件，来自dalao：dalao的git

等待一小段时间后，再运行hexo n "xxxx"来生成md博文时，/source/_posts文件夹内除了xxxx.md文件还有一个同名的文件夹

插入图片

最后在xxxx.md中想引入图片时，先把图片复制到xxxx这个文件夹中，然后只需要在xxxx.md中按照markdown的格式引入图片：

![你想输入的替代文字](xxxx/图片名.jpg)
